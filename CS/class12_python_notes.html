Chapter 1 · Functions

Reusable blocks of code that perform a specific task.

1. What is a Function?

A function is a named block of organized, reusable code that performs a single, related action. Instead of writing the same code again and again, you write it once inside a function and simply call it whenever needed.

Why use functions?

  • Reusability — write once, use many times
  • Modularity — break a big problem into smaller, manageable pieces
  • Readability — code becomes easier to understand and debug
  • Maintainability — fix a bug in one place, it's fixed everywhere the function is used

2. Types of Functions

TypeDescriptionExample
Built-in functionsAlready provided by Python, ready to useprint(), len(), int(), range()
Module/Library functionsDefined inside a module; must import firstmath.sqrt(), random.randint()
User-defined functionsWritten by the programmer using defdef greet(): ...
Lambda (anonymous) functionsSmall, one-line, unnamed functionslambda x: x*x

3. Defining & Calling Functions

def function_name(parameters):
    """optional docstring"""
    # function body
    return value   # optional

A simple example

def greet():
    print("Hello, welcome to Python!")

greet()      # calling the function
greet()      # can call as many times as needed
Output
Hello, welcome to Python! Hello, welcome to Python!
Explanation: def defines the function and gives it a name (greet). Nothing runs until the function is called using greet(). Notice we called it twice — the code inside runs fresh each time, showing reusability.

Function with parameters

def greet(name):
    print("Hello,", name)

greet("Amit")
greet("Riya")
Output
Hello, Amit Hello, Riya
Explanation: name is a parameter — a placeholder variable in the function definition. When we call greet("Amit"), "Amit" is the argument — the actual value passed in. Parameters and arguments are often used interchangeably, but technically: parameter = variable in definition, argument = value at call time.

The docstring

def add(a, b):
    """Returns the sum of two numbers"""
    return a + b

print(add.__doc__)
Output
Returns the sum of two numbers
Explanation: A docstring (triple-quoted string right after def) documents what the function does. It's accessible via function_name.__doc__ or help(function_name) — good practice for every function you write.

4. Arguments & Parameters — All Kinds

4.1 Positional Arguments

def student(name, marks):
    print(name, "scored", marks)

student("Amit", 85)
Output
Amit scored 85
Explanation: Arguments are matched to parameters strictly by position/order. "Amit" goes to name (1st), 85 goes to marks (2nd). Swapping the order changes the meaning.

4.2 Keyword Arguments

def student(name, marks):
    print(name, "scored", marks)

student(marks=85, name="Amit")
Output
Amit scored 85
Explanation: By naming the parameter explicitly (marks=85), order no longer matters — Python matches by name instead of position. Very useful for readability with functions that take many parameters.

4.3 Default Arguments

def student(name, marks=50):
    print(name, "scored", marks)

student("Amit", 90)
student("Riya")          # marks not given, uses default
Output
Amit scored 90 Riya scored 50
Explanation: A default value is used only when the caller doesn't supply that argument. Rule: default parameters must come after non-default ones in the definition — def f(a, b=5) is valid, def f(a=5, b) is a SyntaxError.

4.4 Positional-or-Keyword mixing rule

Order matters when mixing

def student(name, marks=50):
    print(name, marks)

student(name="Amit", 90)   # ERROR
Output
SyntaxError: positional argument follows keyword argument

Once you use a keyword argument, every argument after it must also be a keyword argument.

4.5 Arbitrary Arguments — *args

def total(*numbers):
    print(numbers)
    print(sum(numbers))

total(10, 20, 30)
total(5, 5, 5, 5, 5)
Output
(10, 20, 30) 60 (5, 5, 5, 5, 5) 25
Explanation: *numbers collects any number of positional arguments into a tuple. Use this when you don't know in advance how many values will be passed — like a function that adds any amount of numbers.

4.6 Arbitrary Keyword Arguments — **kwargs

def profile(**details):
    print(details)
    for key, value in details.items():
        print(key, ":", value)

profile(name="Amit", age=17, city="Delhi")
Output
{'name': 'Amit', 'age': 17, 'city': 'Delhi'} name : Amit age : 17 city : Delhi
Explanation: **details collects any number of keyword arguments into a dictionary. Keys become the argument names, values become what was passed.

4.7 Combining all argument types

def demo(a, b, *args, c=10, **kwargs):
    print("a,b:", a, b)
    print("args:", args)
    print("c:", c)
    print("kwargs:", kwargs)

demo(1, 2, 3, 4, 5, c=99, x=100, y=200)
Output
a,b: 1 2 args: (3, 4, 5) c: 99 kwargs: {'x': 100, 'y': 200}
Explanation: Python requires this strict order in a function definition: normal positional parameters → *args → default/keyword parameters → **kwargs. This lets a single function flexibly accept a mix of required, optional, extra positional, and extra keyword arguments — commonly used in real frameworks (e.g. Django, Flask).

4.8 Passing a list/dict directly — unpacking with * and **

def add3(a, b, c):
    return a + b + c

values = [1, 2, 3]
print(add3(*values))     # unpacks list into 3 positional args

data = {"a": 10, "b": 20, "c": 30}
print(add3(**data))      # unpacks dict into keyword args
Output
6 60
Explanation: The */** operators work both ways — inside a function definition they collect arguments, but at a call site they unpack an existing list/dict into separate arguments. This is a favorite "beyond NCERT" exam trick.

5. The return Statement

def square(x):
    return x * x

result = square(5)
print(result)
Output
25
Explanation: return sends a value back to wherever the function was called, and immediately ends the function (any code after return inside that function never runs). A function with no return statement automatically returns None.

Returning multiple values

def stats(numbers):
    return min(numbers), max(numbers), sum(numbers)/len(numbers)

lowest, highest, average = stats([10, 20, 30, 40])
print(lowest, highest, average)
Output
10 40 25.0
Explanation: Python lets you return multiple values separated by commas — they're actually packed into a tuple behind the scenes: (10, 40, 25.0). This is then unpacked into three separate variables on the receiving side.

return vs print — a classic exam confusion

def add_print(a, b):
    print(a + b)

def add_return(a, b):
    return a + b

x = add_print(3, 4)      # prints 7 immediately, but x becomes None
y = add_return(3, 4)     # prints nothing, but y becomes 7

print("x =", x)
print("y =", y)
Output
7 x = None y = 7

print() only displays a value on screen — it does NOT send that value back to the caller. return sends the value back so it can be stored/used further. This distinction is tested very frequently.

6. Scope: Local, Global, and nonlocal

Local scope

def my_func():
    x = 10       # local variable
    print(x)

my_func()
print(x)         # ERROR — x doesn't exist outside the function
Output
10 NameError: name 'x' is not defined
Explanation: A variable created inside a function exists only inside that function — this is its local scope. It's destroyed once the function finishes running.

Global scope

x = 100      # global variable

def show():
    print(x)    # can READ global variable

show()
print(x)
Output
100 100
Explanation: A variable defined outside any function is global — accessible everywhere, including inside functions, as long as you only read it.

The global keyword — modifying a global variable

x = 100

def change():
    global x
    x = 500

change()
print(x)
Output
500
Explanation: Without the global keyword, assigning to x inside a function creates a new local variable instead of changing the global one. The global keyword tells Python: "don't create a new local variable — use and modify the global one."

Common mistake without global

x = 100

def change():
    x = 500     # this creates a NEW local x, doesn't touch global x
    print("Inside:", x)

change()
print("Outside:", x)
Output
Inside: 500 Outside: 100

The global x is untouched — proving that assignment inside a function is local by default unless declared global.

nonlocal keyword nested functions

def outer():
    y = 10
    def inner():
        nonlocal y
        y = 99
    inner()
    print(y)

outer()
Output
99
Explanation: nonlocal is used inside a nested function to modify a variable from its immediately enclosing (but not global) scope. It's like global, but one level up instead of jumping all the way to global scope.

7. Recursion

A function that calls itself is called recursive. Every recursive function needs a base case (a condition to stop) to avoid infinite recursion.

Factorial using recursion

def factorial(n):
    if n == 0:            # base case
        return 1
    else:
        return n * factorial(n - 1)   # recursive case

print(factorial(5))
Output
120
Explanation: factorial(5) calls factorial(4), which calls factorial(3), and so on until factorial(0) returns 1 (base case). Then the results multiply back up: 1×1×2×3×4×5 = 120.
factorial(5) = 5 * factorial(4) = 5 * (4 * factorial(3)) = 5 * (4 * (3 * factorial(2))) = 5 * (4 * (3 * (2 * factorial(1)))) = 5 * (4 * (3 * (2 * (1 * factorial(0))))) = 5 * (4 * (3 * (2 * (1 * 1)))) = 120

Fibonacci using recursion

def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

for i in range(7):
    print(fibonacci(i), end=" ")
Output
0 1 1 2 3 5 8

Recursion vs Iteration

AspectRecursionIteration (loops)
ReadabilityOften shorter, matches mathematical definitionCan be longer but explicit
MemoryUses call stack — more memory, risk of RecursionErrorUsually more memory-efficient
SpeedSlower (function call overhead)Generally faster

Missing base case → infinite recursion

def bad(n):
    return bad(n-1)   # no base case!

bad(5)
Output
RecursionError: maximum recursion depth exceeded

8. Lambda (Anonymous) Functions

square = lambda x: x * x
print(square(5))

add = lambda a, b: a + b
print(add(3, 4))
Output
25 7
Explanation: lambda creates a small, unnamed function in a single line: lambda arguments: expression. There's no return keyword — the expression's result is automatically returned. Lambdas can only contain a single expression, not multiple statements.

Lambda with map(), filter(), and sorted()

nums = [1, 2, 3, 4, 5]

squares = list(map(lambda x: x*x, nums))
print(squares)

evens = list(filter(lambda x: x % 2 == 0, nums))
print(evens)

students = [("Amit", 85), ("Riya", 90), ("John", 70)]
students.sort(key=lambda s: s[1], reverse=True)
print(students)
Output
[1, 4, 9, 16, 25] [2, 4] [('Riya', 90), ('Amit', 85), ('John', 70)]
Explanation: map() applies the lambda to every element and returns a transformed sequence. filter() keeps only elements where the lambda returns True. sorted()/list.sort() use the lambda as a custom key to decide sort order — here sorting students by marks, descending. This combination is one of the most useful patterns in practical Python.

Creating your own module

Explanation: Any .py file can be imported as a module in another file, as long as both are in the same folder (or the module is on Python's search path). This is how large Python projects are organized into multiple files.

10. Beyond Class 12

Closures

def multiplier(n):
    def inner(x):
        return x * n
    return inner

double = multiplier(2)
triple = multiplier(3)
print(double(5))
print(triple(5))
Output
10 15
Explanation: inner "remembers" the value of n from its enclosing function even after multiplier() has finished running — this is a closure. double and triple are independent functions, each carrying their own captured value of n.

Decorators

def my_decorator(func):
    def wrapper():
        print("Before the function runs")
        func()
        print("After the function runs")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()
Output
Before the function runs Hello! After the function runs
Explanation: A decorator is a function that "wraps" another function to add extra behavior without changing its code. @my_decorator is shorthand for say_hello = my_decorator(say_hello). Widely used in real frameworks — e.g. Flask's @app.route().

Function annotations (type hints)

def add(a: int, b: int) -> int:
    return a + b

print(add(3, 4))
print(add.__annotations__)
Output
7 {'a': <class 'int'>, 'b': <class 'int'>, 'return': <class 'int'>}
Explanation: Annotations document expected argument/return types. Python does not enforce them at runtime (you can still pass a string and it won't error) — they're purely for readability and tools like linters/IDEs.

Chapter 2 · Exception Handling

Gracefully dealing with errors that occur while a program is running.

1. Errors vs Exceptions

TypeWhen it happensExample
Syntax ErrorBefore the program runs — invalid Python grammarif x = 5: (missing colon logic, wrong operator)
Exception (Runtime Error)While the program is running — code is syntactically correct but fails during executionDividing by zero, opening a missing file
print(10 / 0)
Output
ZeroDivisionError: division by zero
Explanation: This is syntactically valid Python — it only fails when Python actually tries to execute the division. Without handling, an unhandled exception crashes the entire program.

2. try / except — The Basics

try:
    a = int(input("Enter a number: "))
    b = int(input("Enter another number: "))
    print(a / b)
except ZeroDivisionError:
    print("You cannot divide by zero!")
Sample Output (input 10, then 0)
Enter a number: 10 Enter another number: 0 You cannot divide by zero!
Explanation: Code that might fail goes inside try. If an exception of the matching type occurs, Python immediately jumps to the matching except block instead of crashing — the rest of the try block is skipped.

3. Multiple except Blocks

try:
    a = int(input("Enter numerator: "))
    b = int(input("Enter denominator: "))
    print(a / b)
except ZeroDivisionError:
    print("Cannot divide by zero.")
except ValueError:
    print("Please enter valid integers.")
except Exception as e:
    print("Something else went wrong:", e)
Output (if user enters "abc")
Enter numerator: abc Please enter valid integers.
Explanation: Python checks each except block top to bottom and runs the first one that matches the exception type. The final except Exception as e acts as a catch-all for anything not specifically handled above — e holds the actual error message object.

Catching multiple exception types in one block

try:
    x = int("abc")
except (ValueError, TypeError):
    print("Invalid value or type!")
Output
Invalid value or type!
Explanation: A tuple of exception types in one except lets a single block handle several possible exception types the same way.

4. else and finally

try:
    a = int(input("Enter a number: "))
    b = int(input("Enter another number: "))
    result = a / b
except ZeroDivisionError:
    print("Cannot divide by zero.")
else:
    print("Division successful. Result:", result)
finally:
    print("Execution complete.")
Output (input 10, 2)
Enter a number: 10 Enter another number: 2 Division successful. Result: 5.0 Execution complete.
Explanation: else runs only if no exception occurred in the try block. finally runs no matter what — whether an exception occurred, was handled, or not — commonly used for cleanup actions like closing a file or a database connection.

Order of blocks

The correct order is always: try → except (one or more) → else → finally. else and finally are both optional, and each type of block appears only in this order.

6. The raise Statement

def set_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative")
    print("Age set to", age)

try:
    set_age(-5)
except ValueError as e:
    print("Error:", e)
Output
Error: Age cannot be negative
Explanation: raise lets the programmer deliberately trigger an exception when a certain condition is invalid — even though Python itself wouldn't naturally throw an error there. This is how you build validation logic (e.g. rejecting a negative age).

Re-raising an exception

try:
    try:
        1 / 0
    except ZeroDivisionError:
        print("Logging the error...")
        raise            # re-raises the SAME exception
except ZeroDivisionError:
    print("Caught again in outer block")
Output
Logging the error... Caught again in outer block
Explanation: A bare raise (with nothing after it) inside an except block re-throws the exact same exception, letting you log/handle it partially before passing it further up to another handler.

7. Custom (User-Defined) Exceptions beyond NCERT but examinable

class NegativeMarksError(Exception):
    def __init__(self, message="Marks cannot be negative"):
        self.message = message
        super().__init__(self.message)

def set_marks(marks):
    if marks < 0:
        raise NegativeMarksError()
    print("Marks recorded:", marks)

try:
    set_marks(-10)
except NegativeMarksError as e:
    print("Custom Error:", e)
Output
Custom Error: Marks cannot be negative
Explanation: Custom exceptions are created by defining a new class that inherits from Python's built-in Exception class. This lets you create meaningful, specific error types for your own programs, instead of relying only on generic built-in ones.

8. The assert Statement

def divide(a, b):
    assert b != 0, "Denominator cannot be zero"
    return a / b

print(divide(10, 2))
print(divide(10, 0))
Output
5.0 AssertionError: Denominator cannot be zero
Explanation: assert condition, message checks that a condition is True. If it's False, Python raises an AssertionError with the given message. Commonly used for debugging/testing — checking assumptions that should always be true if the code is correct.

9. Beyond Class 12

Exception chaining with "from"

try:
    try:
        1 / 0
    except ZeroDivisionError as e:
        raise ValueError("Invalid calculation") from e
except ValueError as e:
    print(e)
    print("Original cause:", e.__cause__)
Output
Invalid calculation Original cause: division by zero
Explanation: raise NewError from original_error preserves the original exception as context, so you know both the new error and its root cause — useful in large systems for debugging where an error actually originated.

Exceptions with context managers (relates to file handling)

class SafeDivider:
    def __enter__(self):
        print("Starting safe division block")
        return self
    def __exit__(self, exc_type, exc_value, traceback):
        if exc_type is ZeroDivisionError:
            print("Handled division by zero inside context manager")
            return True     # suppress the exception
        print("Exiting block")

with SafeDivider():
    print(10 / 0)
print("Program continues normally")
Output
Starting safe division block Handled division by zero inside context manager Program continues normally
Explanation: This shows how the with statement (used for files) is actually powered by __enter__/__exit__ methods, and __exit__ can even catch and suppress exceptions — connecting exception handling directly to file handling concepts.

Chapter 3 · File Handling

Reading and writing data to text, binary, and CSV files so it persists beyond a single program run.

1. What is a File?

A file is a named location on secondary storage (hard disk/SSD) used to permanently store data. Unlike variables in a running program (volatile memory, lost when the program ends), files keep data safe even after the program closes.

Why do we need files?

Without files, every program run starts from scratch. Files let programs persist data — student records, transactions, logs, game saves — across multiple runs.

2. Types of Files

TypeDescriptionExample
Text FileReadable characters, lines end with \n.txt, .py
Binary FileRaw bytes, exact copy of data in memory, not directly readable.dat, .jpg
CSV FileText file with comma-separated tabular values.csv

Key exam point

Text files store everything as strings — even 25 becomes the characters '2','5'. Binary files preserve the actual data type — this is why pickle is used to save Python objects directly.

Explanation: Writes a list of strings, but does NOT add \n automatically.

• read(5) → first 5 characters
• readline() → one line with \n

Looping through a file

f = open("data.txt", "r")
for line in f:
    print(line.strip())
f.close()
Output

Explanation: A file object is directly iterable — memory efficient for large files since it reads one line at a time.

Counting lines, words, characters

f = open("data.txt", "r")

Multiple records — list of dictionaries

import pickle

    {"roll": 1, "name": "Amit", "marks": 85},

Chapter 4 · Stack & Queue

Fundamental linear data structures used everywhere in computing — from undo buttons to task scheduling.

1. What is a Stack? (LIFO)

A stack is a linear data structure that follows LIFO — Last In, First Out. The last element added is the first one removed. Think of a stack of plates: you add to the top, and you remove from the top.

PUSH → ← POP ┌───────┐ │ 30 │ ← top ├───────┤ │ 20 │ ├───────┤ │ 10 │ ← bottom └───────┘
OperationMeaning
pushAdd an element to the top of the stack
popRemove and return the top element
peek (top)View the top element without removing it
isEmptyCheck whether the stack has no elements

2. Stack Using a List

stack = []

# push
stack.append(10)
stack.append(20)
stack.append(30)
print("Stack after pushes:", stack)

# peek
print("Top element:", stack[-1])

# pop
removed = stack.pop()
print("Popped:", removed)
print("Stack after pop:", stack)

# isEmpty
print("Is empty?", len(stack) == 0)
Output
Stack after pushes: [10, 20, 30] Top element: 30 Popped: 30 Stack after pop: [10, 20] Is empty? False
Explanation: Python's built-in list works perfectly as a stack: append() is push (adds to the end = "top"), pop() (no argument) removes and returns the last element. Both operations run in O(1) time — very efficient because they operate at the end of the list, not the beginning.

Why NOT use index 0 as "top"?

stack = [10, 20, 30]
stack.pop(0)      # removes from FRONT instead of top — WRONG for a stack

Using pop(0) or insert(0, x) is slow — O(n) — because every other element must shift. Always treat the end of the list as the stack's top.

Popping from an empty stack

stack = []
stack.pop()
Output
IndexError: pop from empty list
Explanation: Always check isEmpty (len(stack) == 0) before popping in real programs, to avoid this crash.

3. Stack Using a Class (proper implementation)

class Stack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        if self.isEmpty():
            print("Stack is empty!")
            return None
        return self.items.pop()

    def peek(self):
        if self.isEmpty():
            print("Stack is empty!")
            return None
        return self.items[-1]

    def isEmpty(self):
        return len(self.items) == 0

    def size(self):
        return len(self.items)

s = Stack()
s.push(10)
s.push(20)
s.push(30)
print("Size:", s.size())
print("Peek:", s.peek())
print("Pop:", s.pop())
print("Size after pop:", s.size())
Output
Size: 3 Peek: 30 Pop: 30 Size after pop: 2
Explanation: Wrapping the list inside a class hides the internal list from outside code, and adds safety checks (like refusing to pop an empty stack). This "object-oriented" style is what real Data Structures courses expect, and is the version examiners often ask you to write.

4. Applications of Stack

Reversing a string

def reverse_string(s):
    stack = list(s)
    reversed_str = ""
    while stack:
        reversed_str += stack.pop()
    return reversed_str

print(reverse_string("PYTHON"))
Output
NOHTYP
Explanation: Every character is pushed onto the stack, then popped off — since pop removes the most recently added item first, the string comes out reversed.

Checking balanced parentheses

def is_balanced(expression):
    stack = []
    pairs = {')': '(', '}': '{', ']': '['}
    for ch in expression:
        if ch in "({[":
            stack.append(ch)
        elif ch in ")}]":
            if not stack or stack.pop() != pairs[ch]:
                return False
    return len(stack) == 0

print(is_balanced("{[()]}"))
print(is_balanced("{[(])}"))
Output
True False
Explanation: Every opening bracket is pushed. Every closing bracket must match the most recently pushed opening bracket (popped from the top) — this is exactly why a stack (LIFO) is the natural data structure for this classic problem, used in real compilers to check code syntax.

Undo functionality (real-world use)

undo_stack = []

def type_text(text, current):
    undo_stack.append(current)
    return current + text

def undo():
    if undo_stack:
        return undo_stack.pop()
    return ""

doc = ""
doc = type_text("Hello", doc)
doc = type_text(" World", doc)
print("Document:", doc)

doc = undo()
print("After undo:", doc)
Output
Document: Hello World After undo: Hello
Explanation: Every text editor's "Undo" (Ctrl+Z) is powered by a stack — each action is pushed before it happens, and undo simply pops the most recent state back.

Evaluating postfix expressions common exam question

def evaluate_postfix(expr):
    stack = []
    for token in expr.split():
        if token.isdigit():
            stack.append(int(token))
        else:
            b = stack.pop()
            a = stack.pop()
            if token == '+': stack.append(a + b)
            elif token == '-': stack.append(a - b)
            elif token == '*': stack.append(a * b)
            elif token == '/': stack.append(a / b)
    return stack.pop()

print(evaluate_postfix("5 3 + 2 *"))
Output
16
Explanation: Postfix (Reverse Polish) notation puts the operator after its operands — "5 3 +" means 5+3. Numbers are pushed onto the stack; when an operator appears, the top two numbers are popped, the operation applied, and the result pushed back. This is how calculators and interpreters evaluate expressions internally.

5. What is a Queue? (FIFO)

A queue follows FIFO — First In, First Out. The first element added is the first one removed. Think of a line/queue at a ticket counter: whoever joins first is served first.

ENQUEUE → → DEQUEUE ┌─────┬─────┬─────┐ │ 10 │ 20 │ 30 │ └─────┴─────┴─────┘ rear front
OperationMeaning
enqueueAdd an element to the rear/back of the queue
dequeueRemove and return the element from the front
peekView the front element without removing it
isEmptyCheck whether the queue has no elements

6. Queue Using a List

queue = []

# enqueue
queue.append(10)
queue.append(20)
queue.append(30)
print("Queue:", queue)

# dequeue
removed = queue.pop(0)
print("Dequeued:", removed)
print("Queue after dequeue:", queue)
Output
Queue: [10, 20, 30] Dequeued: 10 Queue after dequeue: [20, 30]
Explanation: append() adds to the rear, pop(0) removes from the front — implementing FIFO order.

Performance problem with list-based queues

pop(0) is O(n) — every remaining element has to shift one position left in memory. For large queues, this becomes slow. This is exactly why Class 12 syllabus (and real Python code) recommends collections.deque instead of a plain list for queues.

7. Queue Using collections.deque (Recommended)

from collections import deque

queue = deque()
queue.append(10)      # enqueue
queue.append(20)
queue.append(30)
print("Queue:", queue)

removed = queue.popleft()   # dequeue
print("Dequeued:", removed)
print("Queue after dequeue:", queue)
Output
Queue: deque([10, 20, 30]) Dequeued: 10 Queue after dequeue: deque([20, 30])
Explanation: deque ("double-ended queue") allows O(1) fast additions/removals from both ends, unlike a list which is only fast at the end. append() adds to the right, popleft() removes from the left — perfect and efficient for queue behavior.

Full Queue class using deque

from collections import deque

class Queue:
    def __init__(self):
        self.items = deque()

    def enqueue(self, item):
        self.items.append(item)

    def dequeue(self):
        if self.isEmpty():
            print("Queue is empty!")
            return None
        return self.items.popleft()

    def peek(self):
        if self.isEmpty():
            return None
        return self.items[0]

    def isEmpty(self):
        return len(self.items) == 0

q = Queue()
q.enqueue("Amit")
q.enqueue("Riya")
q.enqueue("John")
print("Front:", q.peek())
print("Dequeue:", q.dequeue())
print("Dequeue:", q.dequeue())
Output
Front: Amit Dequeue: Amit Dequeue: Riya

8. Circular Queue important exam topic

A normal queue implemented with a fixed-size array wastes space — once the front moves forward, those slots are never reused. A circular queue wraps around, reusing empty slots at the beginning once the rear reaches the end.

class CircularQueue:
    def __init__(self, size):
        self.size = size
        self.queue = [None] * size
        self.front = self.rear = -1

    def isFull(self):
        return (self.rear + 1) % self.size == self.front

    def isEmpty(self):
        return self.front == -1

    def enqueue(self, item):
        if self.isFull():
            print("Queue is full!")
            return
        if self.isEmpty():
            self.front = 0
        self.rear = (self.rear + 1) % self.size
        self.queue[self.rear] = item

    def dequeue(self):
        if self.isEmpty():
            print("Queue is empty!")
            return None
        item = self.queue[self.front]
        if self.front == self.rear:      # last element removed
            self.front = self.rear = -1
        else:
            self.front = (self.front + 1) % self.size
        return item

cq = CircularQueue(3)
cq.enqueue(10)
cq.enqueue(20)
cq.enqueue(30)
cq.enqueue(40)          # queue full
print("Dequeue:", cq.dequeue())
cq.enqueue(40)          # now there's space, wraps around
print(cq.queue)
Output
Queue is full! Dequeue: 10 [40, 20, 30]
Explanation: The % self.size (modulo) operation is what makes the queue "wrap around" — when rear reaches the last index, the next enqueue goes back to index 0 if it's free. This reuses space that a simple linear queue would otherwise waste.

9. Beyond Class 12

Priority Queue

In a priority queue, elements are removed based on priority, not arrival order — the highest (or lowest) priority element is dequeued first, regardless of when it was added.

import heapq

pq = []
heapq.heappush(pq, (2, "Finish homework"))
heapq.heappush(pq, (1, "Submit assignment"))
heapq.heappush(pq, (3, "Watch a movie"))

while pq:
    priority, task = heapq.heappop(pq)
    print(priority, task)
Output
1 Submit assignment 2 Finish homework 3 Watch a movie
Explanation: Python's heapq module implements a min-heap — the smallest value always comes out first. By pushing (priority, task) tuples, lower priority numbers get dequeued first — exactly how OS task schedulers and hospital emergency rooms prioritize work.

Stack & Queue using a Linked List

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedStack:
    def __init__(self):
        self.top = None

    def push(self, data):
        new_node = Node(data)
        new_node.next = self.top
        self.top = new_node

    def pop(self):
        if self.top is None:
            print("Stack is empty!")
            return None
        popped = self.top.data
        self.top = self.top.next
        return popped

s = LinkedStack()
s.push(10)
s.push(20)
s.push(30)
print(s.pop())
print(s.pop())
Output
30 20
Explanation: Instead of a Python list, each element ("Node") points to the one below it. push creates a new node pointing to the old top, and makes it the new top. pop simply moves top to the next node. This avoids any resizing cost, and is how stacks are implemented at a lower level in languages like C/C++.
